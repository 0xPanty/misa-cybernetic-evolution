## Review Notes

The GitHub baseline commit predates the external-trajectory harness. The final
comparison therefore uses the current harness as a neutral measurement layer:
baseline action/score represents pre-optimization behavior, and calibrated
action/score represents optimized behavior on the same sanitized 867-sample
batch.

The intended review question is not whether this has production authority. It
does not. The intended question is whether the local shadow-only optimization is
positive, stable, and worth keeping as the GitHub branch state.
